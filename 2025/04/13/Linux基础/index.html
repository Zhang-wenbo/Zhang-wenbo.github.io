二.文件和目录操作

pwd查看当前目录位置

ls列出当前文件

touch创建文件

ls *.txt列出所有以 txt 结尾的文件

ls -1竖着列出所有文件

touch .xxx.txt创建隐藏文件，文件名前加.

ls -a列出所有文件，包括隐藏文件

ls -l列出所有文件，包括文件的属性信息

mv 1.txt /top将 1.txt移动到top目录下

ls /top列出top目录下的所有文件

mv 1.txt 2.txt将文件重命名

rm 1.txt删除文件，系统会询问

rm -f 1.txt加参数强制删除

cp 1.txt /opt/2.txt拷贝黏贴重命名文件到指定位置

mkdir创建文件夹

mkdir test{1..5}批量创建文件夹

mkdir -p xx/pp/oo创建层级文件夹

cd ..返回上一级，若返回多级则cd ../../..

history查看历史指令

rm -r zl删除 zl文件夹，只有加-r参数才能识别为目录

rm -rf zl强制删除zl文件夹，包括其子文件

vi编辑文件

cat查看整个文件

tac倒着看整个文件

head -5 1.txt查看1.txt的前 5 行，若不加参数，则默认 10 行

tail -5 1.txt查看1.txt的后 5 行

|管道符，将前面的输出结果作为后面的输入

wc -l 1.txt查看文件行数

wc -c 1.txt查看文件字符数

seq 3 8生成 3 到 8 的序列

seq -w 3 15生成 3 到 15 的等宽序列

grep "pwp" 2.txt模糊匹配包含pwp的行

grep -w "pwp" 2.txt精准 匹配包含pwp的行

grep -wn "pwp" 2.txt匹配包含pwp的行并给出行数

awk '{print $1}' 2.txt默认空格为分割符，去该文本的第一列

awk -F ',' '{print $1}' 2.txt``-F参数可指定分割符，这里指定为,

cat 1.txt | sort查看排序后的文件(默认先数字后字母)

cat 1.txt | sort -n查看排序后的文件(默认先字母后数字)

cat 1.txt | sort | uniq查看排序后的文件，以去重的方式

cat 1.txt | sort | uniq -c查看排序后的文件，以去重的方式，并标记每个元素的出现次数

三.Linux 系统目录功能总结

Linux 系统遵循 文件系统层次结构标准（FHS），以下是常见目录的功能和作用：

1. 核心目录

目录 功能与作用 常见内容示例
/ 根目录，所有目录和文件的起点 所有其他子目录
/bin 基本用户命令（所有用户可用） ls, cp, bash 等二进制文件
/sbin 系统管理员命令（通常需要 root 权限） fdisk, ifconfig, iptables
/boot 启动文件（内核、引导加载程序） vmlinuz, grub/, initramfs
/dev 设备文件（硬件或虚拟设备接口） sda（磁盘）、tty（终端）
/etc 系统配置文件（全局配置） passwd, nginx/, ssh/

2. 用户相关

目录 功能与作用 常见内容示例
/home 普通用户的主目录（每个用户独立子目录） ~/.bashrc, 用户文档、下载等
/root root 用户的主目录 管理员专用配置文件

3. 系统运行

目录 功能与作用 常见内容示例
/proc 虚拟文件系统（内核和进程实时信息） cpuinfo, 进程ID 目录
/sys 系统设备和内核参数（动态配置硬件） 设备驱动、电源管理等
/run 临时运行时数据（系统启动后生成，重启清除） 进程PID、锁文件、套接字

4. 存储与数据

目录 功能与作用 常见内容示例
/var 可变数据（日志、缓存、数据库等） log/, lib/mysql/, cache/
/tmp 临时文件（所有用户可写，重启可能清除） 安装包、临时缓存文件
/usr 用户程序与资源（只读数据，类似 Windows 的 Program Files） bin/, lib/, share/
/opt 第三方软件（可选应用程序） 手动安装的独立软件包

5. 挂载相关

目录 功能与作用 常见内容示例
/mnt 临时挂载点（手动挂载设备） U盘、网络共享临时挂载
/media 自动挂载点（可移动设备自动挂载） CD-ROM、USB 驱动器

6. 其他目录

目录 功能与作用
/lib 系统库文件（/bin 和 /sbin 依赖的库）
/lost+found 文件系统修复后的恢复文件（ext3/4 文件系统特有）
/srv 服务数据（如 Web 服务器的 /srv/www）

7. 注意事项

  1. 权限敏感目录：如 /etc, /root, /sbin 修改需谨慎。
  2. 不要随意删除：系统目录结构对运行至关重要。
  3. 发行版差异：部分目录结构可能因发行版（如 Ubuntu、CentOS）略有不同。

四.用户和用户组

1.用户的添加和删除

cat /etc/passwd查看所有用户

useradd zl添加zl用户

passwd zl为用户设置密码

userdel zl只删除用户，其文件仍被存储

userdel -r zl完全删除该用户及其所有信息

usermod -L zl锁定用户

lchage -l zl查看属性信息

usermod -U zl解锁用户

usermod -h查看可对用户的操作

cat /etc/shadow查看用户的密码信息

w查看当前连接主机的用户信息

cat /etc/group查看用户组信息

groupadd添加组，并不多出用户

useradd -g 组 用户新建用户并分配到指定组

usermod -G 组 用户将用户新加到指定附加组

groupmod -n 新组名 旧组名组 id 并不会随之变化

usermod -g 新组名 用户修改用户主组名
